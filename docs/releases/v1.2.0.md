# v1.2.0 Release Notes

Release date: 2026-03-11

Release title: Durable Local Persistence + Backup/Restore

## Highlights

- Added durable local SQLite store:
  - `Data/state/metrics.db`
- X API snapshots are now append-only:
  - each successful refresh writes a new snapshot row
  - no overwrite of prior snapshots
- Added operational state migration commands:
  - `npm run state:backup`
  - `npm run state:restore -- --from <backup-folder> [--force]`
- Added checksum-verified restore:
  - backup creates `manifest.json` with SHA-256 checksums
  - restore validates checksums before replacing local files

## Why this matters

- API data is no longer lost on restart or reinstall.
- Teams can preserve long-running campaign history locally.
- Recovery between versions is explicit and safer.

## Security notes

- `.env.local` remains local-only and gitignored.
- `Data/state/*` and `Data/backups/*` are gitignored.
- State files use restrictive local permissions where supported.
- API keys are not stored in SQLite snapshots.

## Upgrade guidance

- Pull latest `main`.
- Restart dev server.
- Run a fresh backup before major upgrades:
  - `npm run state:backup`
