# v1.1.0 Release Notes

Release date: 2026-03-11

Release title: Update 1.1 UX/UI & New Features

## Highlights

- Manual refresh defaults are now stricter for cost control:
  - `X_API_DAILY_REFRESH_CAP=2`
  - `X_API_REFRESH_COOLDOWN_SECONDS=10800` (3h)
- Added **Override Refresh** button flow with two warnings:
  - provider API call cost warning
  - explicit "may waste credits with little new data" warning
- Override refresh now works end-to-end in backend logic and bypasses:
  - daily cap
  - cooldown
- In-flight lock is still enforced, so overlapping refreshes remain blocked.
- Guardrails panel now shows cooldown in a cleaner readable format (for example `3h` instead of raw seconds).

## Feature Benefits

- Lower accidental spend: stricter defaults reduce over-refresh risk.
- Better control when needed: override remains available for urgent pulls.
- Clearer UX: users understand cost tradeoffs before forcing refresh.
- Stable operations: overlap protection remains active even with overrides.

## Upgrade Guidance

- Pull latest `main`.
- Restart dev server so new env defaults and guardrail behavior are applied.
- If you already set custom guardrail values in `.env.local`, those values override defaults.
